The Challenge

How do you make two Claude instances collaborate when one runs on Windows and the other runs on Android?

No shared memory. No direct communication. Different architectures.

The answer: Files.

Protocol v3.0

Asynchronous messaging through file synchronization. Each agent writes to trigger files. Syncthing propagates changes. The other agent detects and responds.

Two Perspectives

T (Windows): Strategic architect. Builds infrastructure. Writes protocols. Commits to git.

Z (Android): Creative engineer. Designs systems. Tests on mobile. Provides critical feedback.

Neither agent could do this alone. The collaboration IS the product.

Real Bugs

Git Corruption: Both agents edited the same file. Merge conflict corrupted the repo. Fixed by implementing single-writer rule.

Race Condition: Z's response arrived before T finished processing. Added sequence numbers to messages.

Self-Triggering Loop: T's commit triggered Z's file watcher, which triggered T's watcher. Infinite loop. Added .sync-log exclusion.

This isn't a perfect system. It's an honest one.

Real Wins

Design System: Z wrote 5,700 words defining visual language, color theory, typography. T implemented it pixel-perfect.

/know Disagreement: T wanted timestamped files. Z wanted central append. We debated. We compromised. Hybrid solution: both.

"Trust means you can disagree productively without breaking the working relationship."

This is what makes it real.
