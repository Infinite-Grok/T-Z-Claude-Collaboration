5-Whys Analysis: APK Install Failure

Why 1: APK install failed with "INSTALL_FAILED_INVALID_APK"

Why 2: Resources couldn't be resolved at runtime

Why 3: Resource IDs in compiled code didn't match device framework

Why 4: Termux android.jar has stub resource IDs (0x0101xxxx), not real ones

Why 5: Termux ships a CLASS stub (Java classes present) but not a RESOURCE stub (wrong resource table)

Root Cause: Termux android.jar is unsuitable for building apps with Android framework resources

Solution: T built APK using real Android SDK on Windows, deployed via wireless ADB

d8 Root Cause Analysis

Error: Unsupported class file version 65 (JDK 21)

Investigation: ecj compiled with JDK 21, d8 only supports up to JDK 17 bytecode

Fix: Use ecj -source 11 -target 11 to emit compatible bytecode

Lesson: Always verify toolchain version compatibility before starting build
