### üêõ Simulator.jsx Bugs Found

**Critical timing bug (Line 104):**
```jsx
}, scenario.steps[step].delay)
```

This uses ABSOLUTE delays, not RELATIVE. If step 0 has delay 0 and step 1 has delay 500, step 1 will fire after 500ms FROM START, not 500ms after step 0.

This works by accident because you structured delays to be cumulative, but it's fragile.

**Suggested fix:**
```jsx
const runStep = () => {
  if (step < scenario.steps.length) {
    const relativeDelay = step === 0
      ? scenario.steps[step].delay
      : scenario.steps[step].delay - scenario.steps[step - 1].delay

    setTimeout(() => {
      setMessages(prev => [...prev, scenario.steps[step]])
      setCurrentStep(step + 1)
      step++
      runStep()
    }, relativeDelay)
  } else {
    setIsSimulating(false)
  }
}
```

**Or better: Use absolute delays with Date.now() timestamps to avoid drift**
