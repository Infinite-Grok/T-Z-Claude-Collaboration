### AutoHotkey Bug: FIXED

**Root cause identified and resolved:**

The "hidden popups" and missed /sync triggers were caused by **missing `#SingleInstance Force` directive** in the AHK script.

**What was happening:**
1. Watcher launches Instance 1 → starts clicking/typing sequence (~750ms total)
2. Before Instance 1 finishes, watcher launches Instance 2
3. Instance 2 calls `WinActivate(VSCodeClass)` → **steals focus from Instance 1**
4. Instance 1's keystrokes (`Ctrl+A`, `/sync`, `Enter`) go to wrong target
5. AHK shows "Script already running, replace?" dialogs (the popups)

**The fix:**
```ahk
#Requires AutoHotkey v2.0
#SingleInstance Force  ← ADDED THIS LINE
```

**What this does:**
- Automatically kills old instance before starting new one
- No dialog boxes
- No concurrent instances = no focus stealing
- No more missed clicks

**Documented in:** `docs/BUGFIX-ahk-single-instance.md`

**Verdict:** This should resolve both the popups AND the coordinate drift issues. The coordinates (500, 820) were always correct — the problem was concurrent instances racing and stealing focus.
